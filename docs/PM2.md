# Запуск через PM2 (порт 3013)

## Требования

- Node **>= 20**
- Установленный **PM2** глобально (`npm i -g pm2`) или через `npx` из dev-зависимости проекта

## Переменные окружения

Скопируй `.env.example` → `.env` и заполни как минимум `DATABASE_URL` / `DB_*`, `JWT_ACCESS_SECRET` и остальное.

Порт:

- в **`ecosystem.config.cjs`** по умолчанию **`PORT=3013`**;
- можно переопределить в `.env`: `PORT=3013` или другой порт.

## Сборка и старт

```bash
npm ci
npm run build
npm run pm2:start
```

Проверка:

```bash
curl -sS http://127.0.0.1:3013/api/v1/...
```

## Полезные команды PM2

| Команда | Действие |
|---------|----------|
| `npm run pm2:logs` | логи приложения |
| `npm run pm2:reload` | мягкий перезапуск (после нового `build`) |
| `npm run pm2:stop` | остановка |
| `npm run pm2:delete` | убрать процесс из списка PM2 |

После деплоя новой версии:

```bash
npm run build && npm run pm2:reload
```

## Файлы загрузок

`UPLOAD_DIR` по умолчанию `uploads` относительно **cwd** процесса (корень репозитория). Убедись, что каталог существует и доступен на запись.
